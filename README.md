# Web3 Task Aggregator

PHP + MySQL 任務整合平台：首頁展示任務、管理員與項目方發布任務、會員提交後須審核通過才算完成。任務可設定台灣時間區間、核准名額、結束狀態，以及自訂填寫欄位與選填封面圖。

## 技術棧

PHP（Session、mysqli）、MySQL、Tailwind（CDN）、密碼 bcrypt（`password_hash` / `password_verify`）。

## 功能概覽

| 對象 | 可做什麼 |
|------|----------|
| **訪客** | 瀏覽首頁任務列表（僅摘要）、註冊／登入 |
| **會員** | 看完整說明；以**同頁視窗**填寫任務表單（可 Esc／點背景關閉）；**關閉視窗仍會暫存草稿**（瀏覽器 localStorage）；提交後待審核，駁回可再送 |
| **管理員** | 管理全站任務與使用者角色（含將帳號設為項目方）；審核**全站**待審提交 |
| **項目方** | 管理**自己發布**的任務與提交審核（邏輯與後台類似） |

**任務內容**：公開摘要、完整說明、選填封面圖網址、XP、分類、自訂欄位（單行／多行／網址／Email／勾選，可標示必填或選填）、開始／截止時間、是否已結束、選填核准名額上限。

**首頁任務卡**：顯示時間區間、名額與核准數、是否開放提交；若有 https 封面網址會顯示縮圖。

## 主要頁面

`index.php`（首頁）、`login.php`、`register.php`、`dashboard.php`（後台）、`issuer_portal.php`（項目方）。連線設定在 `db.php`。

## 資料庫與遷移

- **全新安裝**：匯入 `database.sql`（會重建 `group_09`，請先確認可清空）。
- **舊庫升級**（保留資料）：依需求執行  
  - `migrate_p0_p1.sql`：提交狀態、時間窗、通知表等  
  - `migrate_cover_modal.sql`：任務表新增 `cover_image_url`  

匯入後請確認 `db.php` 的資料庫名稱與連線與環境一致（可用環境變數 `DB_HOST`、`DB_USER`、`DB_PASS`、`DB_NAME`）。

## 本機執行

1. 匯入 `database.sql`（或對既有庫跑上述 migration）。  
2. 設定 `db.php`。  
3. 以 XAMPP／WAMP Apache 或 `php -S` 指到專案目錄後開啟對應網址。

## 測試帳號

- 管理員：`admin` / `admin123456`  
- 會員：`member` / `member123456`  
- 項目方：管理員在後台將某帳號角色改為 `issuer` 後，該帳登入會進入項目方中心。

## 備註

正式環境建議使用 HTTPS，並視需求補強 CSRF、登入限制與密碼政策。若資料庫仍為舊版明文密碼，需改為 bcrypt 雜湊後才能用目前登入流程。
