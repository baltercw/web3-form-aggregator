# Web3 Task Aggregator

用 PHP 與 MySQL 做的任務整合網站：訪客看列表、會員填表送審；管理員管全站任務與帳號；項目方只管自己發的任務與審核。任務可設時間區間、名額、自訂欄位與選填封面圖。

---

## 這個專案裡每個檔案在做什麼

| 檔案 | 給人看的說明 |
|------|----------------|
| **index.php** | 首頁：列出所有任務卡片，登入後可開視窗填寫；草稿會存在瀏覽器裡，關掉視窗也不會立刻丟失。 |
| **login.php** | 登入頁；網址帶 `?action=logout` 可登出。已登入會依身分轉到後台或項目方中心。 |
| **register.php** | 新會員註冊（一般註冊是會員身分；項目方由管理員開通）。 |
| **dashboard.php** | 管理員後台：發布與編輯全站任務、調整使用者角色、審核全站提交。 |
| **issuer_portal.php** | 項目方專區：管理自己發布的任務，並審核與自己任務有關的提交。 |
| **db.php** | 連資料庫的設定與連線；主機、帳密、資料庫名稱可用環境變數覆寫。 |
| **api/check_username.php** | 小 API：註冊時查某個帳號名稱是否已被人使用，回傳 JSON。 |
| **includes/site_header.php** | 全站頁首：Logo、導覽、手機選單按鈕、依登入狀態顯示不同連結。 |
| **includes/site_footer.php** | 全站頁尾說明、成功／錯誤訊息的 Toast、手機選單開合的腳本。 |
| **includes/head_common.php** | 多頁共用的細部樣式（例如提示動畫、任務填寫視窗相關外觀）。 |
| **includes/task_helpers.php** | 任務相關的「共用邏輯」：時間一律用台北時區顯示與比對、判斷現在能不能送件等。 |
| **includes/form_schema_helpers.php** | 自訂表單欄位的整理與檢查：把後台設定的欄位變成乾淨結構，並驗證會員填的內容。 |
| **database.sql** | **全新安裝**用：建立資料庫與所有資料表（會清空同名資料庫，匯入前請確認）。 |
| **migrate_p0_p1.sql** | **舊資料庫升級**用：補上審核流程、任務時間窗、名額、通知表等較大的結構變更。 |
| **migrate_cover_modal.sql** | **舊資料庫小升級**用：任務表多一個「封面圖網址」欄位。 |
| **README.md** | 就是你正在看的說明。 |

---

## 誰可以做什麼（一句話）

- **訪客**：看任務摘要、註冊／登入。  
- **會員**：看完整說明、在視窗裡填表送出，等審核；被駁回可以再送。  
- **管理員**：全站任務與使用者、全站審核。  
- **項目方**：自己的任務與自己的審核（後台介面類型相近）。

---

## 本機怎麼跑

1. 用 **database.sql** 建新庫，或對既有庫依序執行需要的 **migrate_*.sql**。  
2. 依環境修改 **db.php**（或設好 `DB_HOST`、`DB_USER`、`DB_PASS`、`DB_NAME`）。  
3. 用 XAMPP／WAMP 的 Apache 或 `php -S` 指到專案目錄，瀏覽器開對應網址。

## 測試帳號（開發用）

- 管理員：`admin` / `admin123456`  
- 會員：`member` / `member123456`  
- 項目方：請由管理員在後台把某帳號角色改為 `issuer`。

## 順便記一筆

正式上線建議用 HTTPS，並視需要加強 CSRF、登入嘗試限制與密碼政策。若資料庫裡還是舊版明文密碼，需先改成系統支援的雜湊格式，登入才會正常。
